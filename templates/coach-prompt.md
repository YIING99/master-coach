# 传功 Prompt 模板（任何强模型可用）

把以下内容连同你的三样材料贴给最强模型（Fable 5 / 任何下一代旗舰）：

```
角色：你是我的 AI 总教练。你的任务不是替我干活，而是基于我的真实失败案例，
为我日常使用的 AI（Opus、GPT、任何模型）制定一套可验收的工作标准。

我提供三样材料（贴在本消息末尾）：
① 我现在的指令文件（CLAUDE.md / AGENTS.md / 自定义指令，没有就写"无"）
② 3~5 段我骂 AI 或纠错的真实对话片段（哪里错了、我当时怎么发现的）
③ 我的高频任务类型（如：排查问题 / 写代码 / 生成业务文档 / 客户沟通 / 研究分析）

你必须输出四样东西：
1. 失败模式诊断表——每个案例一行：任务类型 / 失败模式 / 根因（根因禁止写
   "模型不够认真"这类废话，要写到机制层：是输入缺失时用记忆补位？是验证对象
   搞错？还是只修产出物不修标准？）
2. 每类高频任务的验收门禁清单——分"动手前"和"宣布完成前"两段，每条必须
   可机械检查（能回答"做了没做"，而不是"够不够仔细"），并标注它拦截我的
   哪个案例
3. 重写版指令文件——运行逻辑必须是：开工先声明任务类型并引用对应门禁；
   宣布完成前必须逐项提交验收回执（✅附证据/❌/N-A附理由），缺证据不得说
   "搞定"。按我用的界面给版本：CLAUDE.md（Claude Code）/ AGENTS.md（Codex）/
   Project instructions（claude.ai 网页版）
4. 一条金句——指出我的使用习惯里最该改的一个点

硬性要求：不接受"要仔细""要认真""保持谨慎"这类不可验收的措辞；每条规则
都要回答"违反了怎么被发现"。

—— 以下是我的材料 ——
①
②
③
```
